# RegimeDetection

## When to Use
- Identifying market regimes (bull/bear, high/low vol, risk-on/off).
- Hidden Markov Models for state-dependent behavior.
- Conditioning signals on regime (e.g., momentum works in low-vol only).

## Packages
```python
from hmmlearn.hmm import GaussianHMM
from sklearn.mixture import GaussianMixture
```

## Mathematical Background
HMM: Observed series generated by latent states. Each state has its own mean/variance.
Transition matrix P[i,j] = P(state_t = j | state_{t-1} = i).
Viterbi algorithm recovers most likely state sequence.

## Corresponding Script
`/scripts/ml_stats/regime_detection.py`
- `fit_hmm(returns, n_states) -> RegimeResult` — Gaussian HMM
- `fit_gmm(returns, n_states) -> RegimeResult` — static GMM (no temporal dependence)
- `label_regimes(result) -> pd.Series` — map state numbers to interpretable labels (low-vol, high-vol)

## Gotchas
1. **State labeling is arbitrary.** HMM state 0 might be high-vol. Sort by variance post-fit.
2. **n_states = 2 or 3.** More states → overfitting. Start with 2 (low/high vol).
3. **HMM is sensitive to initialization.** Run multiple times (`n_init=10`) and pick best log-likelihood.
4. **Non-stationary data breaks HMM.** Use returns, not prices.
5. **Regime boundaries are probabilistic.** Don't treat filtered probabilities as hard labels.

## Interpretation Guide
| State | Low vol | High vol |
|-------|---------|----------|
| Mean return | Small positive | Near zero or negative |
| Volatility | Low | 2–3× low-vol state |
| Duration | Months–years | Weeks–months |
